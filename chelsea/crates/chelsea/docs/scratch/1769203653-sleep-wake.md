### R-03847-31405-22784-63423-49651-07496-19884-41962
The external API may allow a 'Booting' VM to be put to Sleep, if and only if the `skip_wait_boot` param (default false) is false, in which case it will wait for the VM to enter the 'Booting' state before proceeding with the operation as normal.
```sh
vm_id=$(./api.sh new | jq -r .vm_id)
echo "Attempting to sleep VM without waiting for boot (failure expected)"
./api.sh sleep $vm_id --skip-wait-boot
echo "VM state: $(./api.sh status $vm_id | jq -r .state)"
echo "Sleeping VM while waiting for boot (default when no query param)"
./api.sh sleep $vm_id
echo "VM state: $(./api.sh status $vm_id | jq -r .state)"
```

### R-62026-61944-40621-46513-17994-04169-19652-27432

```sh

image_id1=$(sqlite3 $db "SELECT image_name FROM ceph_vm_volume WHERE id = (SELECT vm_volume_id FROM vm WHERE id = '$vm_id');")
echo "Expected one mapped volume with image ID $image_id1:"
rbd showmapped | grep $image_id1
./api.sh sleep $vm_id
echo "Expected exactly one RBD entry"
rbd --id chelsea ls -l | grep $image_id1

rbd showmapped | grep $image_id1

echo "Expected exactly one RBD entry"
```

### R-19457-35508-64869-09253-42494-60138-32694-50591
Upon Waking a VM, the snapshot created when it was put to Sleep will be internally marked as used, and associated resources may be considered for freeing.  
Currently, this is done immediately, rather than being "marked" for garbage colleciton.
```sh
pg_pass=opensesame
pg_user=postgres
pg_db=vers
export pg="postgresql://${pg_user}:${pg_pass}@127.0.0.1:5432/${pg_db}?sslmode=disable"

bucket=vers-sleep-snapshots-dev--use1-az4--x-s3 

vm_id=$(./api.sh new | jq -r .vm_id)
./api.sh sleep $vm_id
echo "VM status: $(./api.sh status $vm_id)"
echo "Expected sleep snapshot record:"
psql $pg -c "SELECT vm_id, volume_sleep_snapshot FROM chelsea.sleep_snapshot WHERE vm_id = '$vm_id'"
echo "Expected 2 S3 files:"
aws s3 ls s3://$bucket/$vm_id/
./api.sh wake $vm_id
echo "VM status: $(./api.sh status $vm_id)"
echo "Expected 0 sleep snapshot records:"
psql $pg -c "SELECT vm_id, volume_sleep_snapshot FROM chelsea.sleep_snapshot WHERE vm_id = '$vm_id'"
echo "Expected 0 S3 files:"
aws s3 ls s3://$bucket/$vm_id/
```

### R-13790-40027-17422-16765-07334-42062-07089-34692

(Partially) tested by chelsea-105.0. (The test lacks assertions for running state, and only has an assertion for FS state.)

## Unmet requirements

### R-18641-06866-22363-08646-49220-58287-42489-64904

See #806

### R-52569-51782-30849-06194-08440-59755-05660-50941

The snapshot associated with a Sleeping VM must never be considered for deletion.
