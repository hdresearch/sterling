# About
This is maybe not the most elegantly-named or elegantly-placed library. But there was a need to do this, because `chelsea_server` is now listening on a Wireguard interface. This means that the `POST /api/vm/{vm_id}/notify` endpoint became inaccessible to VMs. In order to ensure the notify ready mechanism still works, I've gone ahead and split out this particular endpoint onto its own webserver. I have no strong opinions about where this lands eventually, but the important thing is that it remains accessible to the VMs through the host networking stack at the expected location; as of October 15, 2025, that location is `192.168.0.0/api/vm/{vm_id}/notify`. (Note that the lack of port implies port 80). If this changes, then the systemd service being inserted into our rootfses must be changed to align.
